Introduction

As research on blockchain and its applications currently focuses too much on the creation of new blockchains and cryptocurrencies, there have been a large number of diverse blockchain networks. These networks operate independently and separately from each other; each aims to solve a limited set of problems. Meanwhile, the real world problems are often so complex and diverse that the blockchains need to exchange data and collaborate well with each other. This gap makes blockchain technology not widely used in real world applications. Even simple functionalities, e.g., identifying a crypto-person or executing smart contracts could not be performed on multiple blockchains.

ToVChain is the world‚Äôs first hyper-linked datachain protocol that links both data and functionalities across distributed ledgers to drastically increase the adoption of blockchain technologies in real problems. While other projects and research deal with simple data interoperability among multiple blockchain networks, we define the new term ‚Äúsemantic interoperability‚Äù and offer a unique solution for comprehensive and profound interoperability in both data and functionalities. Three distinctive features that differentiate our approach from the others are as follows: (i) the establishment of semantic links among the data elements in blockchain networks in a trustable, verifiable and immutable manner, (ii) the automatic migration of a whole DApp including all of its components and data from a blockchain network to another network, and (2) the automatic code generation of DApps that have components (e.g., smart contracts, processors) running distributedly on multiple network blockchains to perform particular business processes.

A technical report [2] from the U.S. National Institute of Standards and Technology (NIST) defines blockchain interoperability as ‚Äúa composition of distinguishable blockchain systems, each representing a unique distributed data ledger, where atomic transaction execution may span multiple heterogeneous blockchain systems, and where data recorded in one blockchain are reachable, verifiable, and referable by another possibly foreign transaction in a semantically compatible manner‚Äù. The definition focuses on the data level only.

To deal with the blockchain data interoperability problem, sidechain, abstraction layer, and hub/adaptor have been three widely used techniques. Sidechain is the small blockchain network, which runs parallely with the main blockchain to synchronize data and increase the performance for users. Abstraction layer (message protocols) provides an abstraction layer for gathering and delivering data among different blockchain. Hub/adaptor is the coupling means for blockchains that enables transferring data from a particular one to another.

These three techniques focus on data interoperability, which is based on data exchange protocols, in order to create a vision of the Internet of blockchains. There have been no projects or techniques ever considered the semantics of data interoperability as mentioned in the NIST definition. Moreover, interoperability at data level only would never make the dream of ‚ÄúInternet of blockchains‚Äù come true.

Meanwhile, the gray colour expresses the existing solutions today. Although, in the way of resolving the blockchain interoperability problem, we still need to deal with these gray parts toward to build up a complete blockchain interoperability product. To this end, ToVChain also owns the feature of data exchange between multiple networks, however, we increase the value of data through semantic links. In addition, we focus on user identification through Cross-KYC protocol to identify and establish connections for diverse IDs of the same user on different systems. Finally, Cross-DApp protocol can not only migrate the whole source code and data of a DApp to a new blockchain network, but also allow developers to combine different functionalities running on multiple blockchains into the same DApp.
