System Overview

There have been several solutions to swap cryptocurrencies among blockchain networks. This swapping mechanism can be considered as a simple type of linking blockchain data. ToVChain draws a broader vision of linking various kinds of blockchain data and functionalities to leverage their value. Our vision is to:

  • Aggregate data on each individual and thereby provide a complete view of them. The aggregated data can be used in various manner, e.g., authentication, credit score evaluation.
  • Establish links among data elements used in DApp applications to not only enhance reliability but also considerably increase its value. For example, all operations of a vehicle might be stored on different blockchain networks in a separated manner. A traceability system records the vehicle location history on a blockchain network, while an electronic toll collection system also records the vehicleâ€™s images and information on another blockchain network when it is passing through a station. Once those records are linked to each other, they become more trustable and provide more information on the vehicle operations.
  • Ensure the transparency and the origin of data. For example, an image is stored and shared among an IPFS network. Users of this network can make a copy of the image and put it on the other Arweave storage system. Linking the two images allows us to annotate that they are the same, and the first one in the IPFS network is the original.

The central of the architecture is our Cross-chain Knowledge Graph. It is a distributed ledger deployed on a number of computers. A set of validators, which implement the ToVChain protocols, contain the Data Monitor component to collect data stored in/used by/referred by blockchain networks in real-time. The collected data is then analyzed, aggregated and linked to each other. The data links need to be verified and validated before being written into the ledger as ToVChain transactions. All transactions are used to build up the cross-chain knowledge graph.

Through a Query and Inference Engine, members of ToVChain Network (DApp, Oracle, etc.) get access to the graph via our three blockchain-based protocols. They can query data across multiple blockchain networks to get more insights, swap digital assets stored in different networks, or perform interoperability transactions.

ToVChain can charge DApps and Oracles transaction fees to (1) write hyper-linked datachain on the ledger, (2) issue a certification of cross-chain relationship between two entities; (3) transfer data across multiple chains for interoperability transactions such as swapping tokens, (3) automatically migrate the DApps from a network to another one
