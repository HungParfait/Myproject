Terminology

ToVChain#

ToVChain is a hyper-linked datachain protocol designed to provide semantic interoperability among data elements and functionalities of heterogeneous blockchain networks.

Data interoperability#

This term refers to the limited interoperability among blockchain networks at data level, which is adopted by many projects and research.

Semantic interoperability#

This is our innovative and ambitious term, denoting the comprehensive and profound interoperability in both data and functionalities among multiple blockchain networks. It is achieved through our four fundamental protocols, i.e., data exchange, data linking, cross-KYC and cross-DApp protocols.

Datachain#

Datachain is a chain of related data. Data is always a crucial and invaluable element in any system. Using blockchain technology, data is traceable, immutable, and thus can be trusted and verified. Though blockchains store data in blocks that are chained together, neither the data in a single blockchain or the data across multiple blockchain is semantically linked to each other. There have been no blockchain networks that annotate and make use of the sameAs relationship between two addresses, or the colleague relationship between two track drivers in a blockchain application for transportation networks. The goal of ToVChain is to leverage the value of blockchain data by establishing semantic links among data elements stored in both single and multiple blockchain networks. We categorize data into two main groups:

  • Financial data refers to the cryptocurrencies (i.e., coin, token) and the transactions to lend, borrow, and transfer those assets in the blockchain networks.
  • Non-financial data refers to any other types of data stored/referred by/used in the blockchain networks. Examples are diploma records, medical records, vehicle records, transportation data, insurance data, and supply chain data.

Hyper-linked datachain#

This term is inspired by the term of Linked Data, which was introduced by Sir Tim Berners-Lee, the inventor of WWW, in 2008. Linked data refers to the vision that data on the Web are interlinked to each other in order to provide semantic relationships among them and thus foster data integration. In a similar manner, we aim to identify and represent possible interconnections among data elements stored on available networks. Examples of such semantic relationships are sameAs, type, createdBy, providedBy.

Protocol#

Protocol refers to a set of rules that allows two or more participants within the system to communicate effectively with each other. These rules define syntax, semantics, format, synchronization mechanisms, and possibly contain means to detect and correct errors. Many protocols such as HTTP (Hypertext Transfer Protocol), TCP (Transmission Control Protocol), IP (Internet Protocol), IMAP (Internet Message Access Protocol) have been developed and widely used today.

Our Hyper-linked datachain protocol plays a similar role. It defines rules to exchange, link, and verify data among diverse blockchain networks. The protocol needs to satisfy many requirements on security, connectivity, consensus, and authentication of transactions. The Hyper-linked datachain protocol consists of data exchange protocol, data linking protocol, cross-KYC protocol and cross-DApp protocol.

Cross-chain#

The term mentions the interoperability among isolated blockchain networks. Existing networks (e.g., Bitcoin, Ethereum, and Hyperledger Sawtooth) make use of its own fundamental foundations with diverse types of consensus algorithms, network topologies, and smart contracts. Those networks hence can not directly communicate and transfer data to each other. To address this issue, we create a mediated communication standard and establish a mechanism to transform data from a specific network to this common standard.

Cross-KYC (Know your customer)#

Cross-KYC provides a detailed and complete view on a blockchain entity, which can be a person, an organization, or other kinds of data elements. ToVChain is responsible for identifying, authenticating, and linking different person identities established in multiple on-chain and off-chain data sources. This allows us to annotate the address of one person on multiple blockchain networks and evaluate the total assets of that person on those networks.

Cross-data#

This term shows ToVChainâ€™s broader vision when any types of data (such as identifications, cryptocurrencies, documents, etc.) stored on on-chain and off-chain data sources can be linked to each other to enhance their values. This forms a cross-chain knowledge graph that continuously expands. Each interconnection needs to be validated and verified by our ToVChain protocol before being added into the graph. The cross-data term encompasses both data exchange and data linking terms

Cross-DApp#

This term refers to our two innovative techniques, i.e., (1) automatic migration of a whole DApp including all of its components and data from a blockchain network to another network, and (2) automatic code generation of DApps that have components (e.g., smart contracts, processors) running distributedly on multiple network blockchains to perform specific business logic. Those components work under a coordinator, which can also be generated automatically; all are supported to interact with other utility applications and services like decentralized file system storages (e.g., IPFS or Arweave).
